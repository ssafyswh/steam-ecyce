# 🚀 Steam Ecyce: AI 기반 개인화 게임 분석 & 커뮤니티 플랫폼

## 1. 프로젝트 개요

### 1-1. 프로젝트 소개

**Steam Ecyce**는 스팀(Steam)의 방대한 데이터를 AI로 분석하여, 유저에게 **직관적인 게임 경험**과 **소통의 장**을 제공하는 플랫폼입니다. 단순한 게임 정보 조회를 넘어, AI가 유저의 플레이 스타일을 분석하고, 복잡한 리뷰를 읽기 쉬운 칼럼 형태로 요약해 줍니다.

### 1-2. 문제 정의 및 해결 목표

| 카테고리 | 문제 정의 (Pain Point) | Steam Ecyce의 해결책 (Solution) |
| --- | --- | --- |
| **검색 시스템** | 유사 게임 추천 기능 부재 등 스팀 검색 불편 (오타 입력 시 검색 실패) | AI 기반 대체 검색 기능 제공 |
| **리뷰 시스템** | 리뷰 컨텐츠의 **요약 기능이 부재**하여 유용한 리뷰를 걸러내기 어려움 | AI 기반 리뷰 요약 및 구조화 |
| **추천 알고리즘** | 게임 추천 알고리즘 미흡, 개인화 수준 부족 | **플레이 게임 기반 게임 추천** 및 **플레이한 게임 이상형 월드컵** |
| **컨텐츠** | 단순 라이브러리 목록 | **플레이한 게임 이상형 월드컵** 기능 |
---

## 2. 핵심 기능 및 차별점

### 🌟 1. AI 기반 심층 분석

* **통합 마이페이지**: 유저의 스팀 라이브러리를 분석하여, 단순 통계가 아닌 **"당신은 어떤 게이머인가"**를 설명하는 자연어 리포트를 제공함과 동시에 성향에 맞는 게임을 추천합니다.
* **유기적 데이터 연동**: 이상형 월드컵에서 우승한 게임을 **'최애 게임'**으로 프로필에 등록하고, 클릭 시 상세 정보로 즉시 연결됩니다.

### 📝 2. AI 리뷰 큐레이션

* **칼럼니스트 페르소나**: 수천 개의 리뷰를 나열하는 대신, AI가 **전문 칼럼니스트의 어조**로 장단점을 종합한 하나의 완성된 글을 작성합니다.

### 🤝 3. 스마트 검색 시스템 

* **유연한 검색 기능 제공**: 게임 제목 검색 시 정확하게 일치하는 데이터가 없을 경우 AI가 **한/영 변환, 오탈자** 등을 고려하여 사용자의 의도를 파악하는데 도움을 줍니다.

### 🤝 4. 이상형 월드컵과 결과 공유 커뮤니티

* **사용자의 라이브러리를 이용하여 새로운 컨텐츠를 제공**: 사용자가 플레이한 게임을 기반으로 **이상형 월드컵**을 진행하고, 그 결과를 공유할 수 있는 커뮤니티를 제공합니다.

## 3. 상세 요구사항

### 3-1. 기능적 요구사항 (Functional)

| ID | 구분 | 상세 내용 | 비고 |
| --- | --- | --- | --- |
| **F01** | **인증** | Steam OpenID 로그인 및 JWT/Session 기반 인증 관리 | `api/auth/` |
| **F02** | **데이터 동기화** | Steam Web API 연동을 통한 라이브러리 및 플레이 타임 실시간 동기화 |  |
| **F03** | **마이페이지** | **[통합]** 유저 정보, AI 성향 분석 텍스트, 최애 게임 정보를 한눈에 볼 수 있는 대시보드 | `api/auth/user/mypage/` |
| **F04** | **게임 상세** | 게임 기본 정보 및 **AI 요약 칼럼** 제공 | 무료 게임 포함 |
| **F05** | **커뮤니티** | 게시글 작성(이미지 첨부), 조회, 상세 보기 기능 | `FormData` 활용 |
| **F06** | **월드컵** | 보유 게임 기반 이상형 월드컵 진행 및 결과 프로필 반영 |  |

### 3-2. 비기능적 요구사항 (Non-Functional)

* **Usability**: 로그인 상태에 따라 '라이브러리' 등 개인화 메뉴를 조건부 렌더링하여 UX 혼선 방지.
* **Reliability**: 존재하지 않는 페이지 접근 시 라우팅 에러 처리 및 무료 게임 데이터 예외 처리 로직 적용.
* **Design**: 하얀색을 기반으로 한 일관적인 테마의 UI 디자인 제공.

## 4. API 명세서 (Current Implementation)

* **Base URL**: 기능별 분리 (`/api/auth/`, `/api/v1/` 등)

### 🔐 1. 인증 및 유저 (Auth & User)

| 기능 | Method | Endpoint | 설명 |
| --- | --- | --- | --- |
| **스팀 로그인** | `POST` | `/api/auth/login/` | 스팀 계정 인증 및 토큰 발급 |
| **통합 마이페이지** | `GET` | `/api/auth/user/mypage/` | **[핵심]** 유저 프로필 + AI 분석(Gamer Type) + 최애 게임 데이터 통합 반환 |

### 🎮 2. 게임 및 분석 (Games & Analysis)

| 기능 | Method | Endpoint | 설명 |
| --- | --- | --- | --- |
| **게임 상세/AI요약** | `GET` | `/api/v1/games/{appid}/` | 게임 상세 정보 및 AI 리뷰 요약문 반환 (30분 캐싱) |
| **리뷰 분석 요청** | `POST` | `/api/v1/games/{appid}/analyze/` | AI 리뷰 분석 강제 실행 및 갱신 |

### 💬 3. 커뮤니티 (Community)

| 기능 | Method | Endpoint | 설명 |
| --- | --- | --- | --- |
| **게시글 목록** | `GET` | `/community/articles/` | 전체 게시글 조회 |
| **게시글 작성** | `POST` | `/community/articles/` | 제목, 내용, 이미지(Files) 업로드 |
| **게시글 상세** | `GET` | `/community/articles/{id}/` | 특정 게시글 상세 내용 조회 |
